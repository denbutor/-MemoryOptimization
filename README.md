Порівняння з попередніми версіями:
---
1. Проблема попередніх версій - споживання пам'яті. Спочатку завантажувався увесь файл у пам'ять (readAllBytes), створювалася ще одна копія файлу як String, а потім створювався величезний масив String[] words елементів, що містив кожне слово з тексту.
2. Files.lines(Paths.get(FILE_PATH)): Цей метод не читає весь файл одразу. Він створює потік (Stream), який читає файл рядок за рядком у міру обробки.
3. Collectors.groupingBy(...): Це єдине місце, де накопичуються дані, але воно зберігає в пам'яті лише унікальні слова та їх лічильники (Map<String, Long>). Це в рази менше, ніж зберігати кожне слово з файлу.
4. Кожен рядок очищується, ділиться на слова, і ці слова одразу ж потрапляють у колектор (groupingBy).
5. Паралельні потоки використовують додаткову пам'ять для буферизації та управління потоками, тому треба прибирати .parallel(). Послідовний потік використовує менше пам'яті.
